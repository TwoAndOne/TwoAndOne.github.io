搭Hexo+Github博客之种种磨难

Posted on 2020-08-19 Edited on 2021-03-28 In tech

搭一个hexo+GitHub的博客，外加换主题，花了一天时间。 唉，本来预想是两小时搞完，但总是不尽人意。 众所周知，学习任何计算机知识，第一步是一道坎——环境配置。

总结: 很多问题的出现，可以归结于自己的强迫症。

1.安装完node.js后，非要去改npm全局包默认存储路径(C盘！你别告诉我你能忍?)

结果改完环境变量，最后版本还不对，又重新卸载安装。重安后，不小心勾选了自动安装相关tools，然后它竟然开始下载Python3.8、visualstudio2019……天哪！我当即Ctrl+C终止，又不辞辛苦卸载，重安。对，你猜的没错，重安后，我还是继续慢慢改环境变量，更改存储路径。

2.接着是hexo的下载，这个可以用npm下载，没错，又遇到问题了！ timeout超时

由于hexo是托管在GitHub上的，国内访问GitHub特别喜欢超时。

所以我曾经更改了hosts文件，希望访问快一点，没想到就是这个原因，导致我无法成功下载hexo。

解决方法是把hosts文件里关于GitHub的全删掉。我兴致冲冲找到开文件路径，准备大删一番，结果你能想到，自然是铩羽而归(总是这样，我快习惯了）。

因为前几天电脑网络不稳定，试过能试的办法，我猜想是不是中了病毒，便下了个360来杀毒，又因为我修改了hosts文件，它把hosts文件锁了，导致我无法打开。

命运啊，总是这么循环🔁且无奈，前世种的因，今生就得果。曹刿曾曰:一鼓作气，再而衰，三而竭。现在的我竭得不能再竭了。

打开360杀毒软件，借来齐天大圣的火眼金睛，终于找到添加白名单的按钮(实在是不敢乱点，真被吓到了，连正常软件都遇到诸多问题，软件中的流氓360那可是惹都不敢惹，动不动全家桶警告）。

hosts修改成功，来吧，下载hexo。

3.这是最简单的问题也是耗时最长的问题，一切都根源都来自我内心深处对美的追求(该死的强迫症，别为自己找借口！）。

如果不换主题，这三四个小时完全可以用来玩耍，所以可恶啊！

找bug的过程就不一一赘述了，纯粹是个搜商问题，再一次感慨Google的强大，连GitHub里某个issue的评论都能搜出来。
