win10在无线网卡连接下开启热点

Posted on 2020-05-03 Edited on 2021-03-28 In tech

  1. win+r 打开命令提示符框（管理员模式）

  2. 在cmd中输入 netsh wlan show drivers 检查无线网卡是否已插入

  3. 继续输入 netsh wlan hostednetwork mode=allow ssid=Yus.hotpot key=password 设置热点名与密码

  4. 输入 netsh wlan start hostednetwork 开启WIFI热点

  5. 打开 控制面板\网络和 Internet\网络连接 在“以太网2”右键属性，点击允许其他网络xxxx，家庭网络连接选择本地连接13（这里你的编号是下图中你设置了热点名的那一个连接）

  6. 手机打开WLAN进行连接

spring配置文件读取顺序导致数据库连接的bug

Posted on 2020-04-15 Edited on 2021-03-28 In tech

java.sql.SQLException: Access denied for user ‘root’@’localhost’ (using password: YES)

在连接数据库的时候，使用yaml文件配置了数据库相关的内容，比如用户名，密码，url，驱动，测试连接时却出错了，拒绝访问。

首先检查了MySQL的服务是否开启，接着检查url是否正确，比如用以前连过的url进行连接，但是都不行。

去网上搜索，搜到的答案前篇一律都是改yaml文件的比如用户名加引号，密码不能0开头，但是都没有解决。

后来一不小心点开application.properties文件，发现这个文件里也这些配置，不够用户名和密码是星号，瞬间悟了。

去了解之后发现，spring默认优先读取yaml文件，再读取properties文件，然后我的配置就会被properties文件所覆盖。

理解@Bean

Posted on 2020-04-12 Edited on 2021-03-28 In tech

首先看一下源码：（建议配合“理解注解一起使用”）

从头到尾梳理一遍：

@Target({ElementType.METHOD, ElementType.ANNOTATION_TYPE})

表示可以在方法和注解上

@Retention(RetentionPolicy.RUNTIME)

表示运行时可以通过反射访问

表示可以定义别名，返回值是数组，所以可以有多个，默认和方法名相同。

下面是官方文档的介绍：

Indicates that a method produces a bean to be managed by the Spring container.
Overview
The names and semantics of the attributes to this annotation are intentionally similar to those of the element in the Spring XML schema. For example:
@Bean
public MyBean myBean() {
// instantiate and configure MyBean obj
return obj;
}

翻译一下：

指出某个方法可以生产一个被Spring容器管理的bean

这个注解的属性和语言和Spring XML文件中和<bean/>标签类似。

总结

被@Bean注解的方法，JVM会利用反射生产出一个和方法返回值类型一样的实例对象，并且由Spring容器管理。

理解Annotation

Posted on 2020-04-06 Edited on 2021-03-28 In tech

什么时候引入？

JDK5.0

哪些地方可以用

Java 语言中的类、方法、变量、参数和包等

怎么用

在编译器生成类文件的时候，可以通过反射获取注解的内容，把它嵌入到字节码中 。Java虚拟机可以保留注解内容，在运行时获取注解的内容。

元注解

  • @Retention - 标识这个注解怎么保存，是只在代码中，还是编入class文件中，或者是在运行时可以通过反射访问。
  • @Documented - 标记这些注解是否包含在用户文档中。
  • @Target - 标记这个注解应该是哪种 Java 成员。
  • @Inherited - 标记这个注解是继承于哪个注解类(默认 注解并没有继承于任何子类)

源码

Annotation.java

看看注解的存活时期的注解

Retention.java

神奇，自己注解自己，好，我们点进RetentionPolicy中的看看

翻译一下（四级水平，见谅）

SOURCE:注解会在编译的时候丢弃，也就是说就在编译时起作用，检查检查代码，比如著名的Override

CLASS:注解会被编译器记录成字节码文件，但是不会存活到VM运行时，这是默认的

RUNTIME:注解会被编译器记录成字节码文件，并且在存活到虚拟机运行时，所以他们会被反射的读取。

看看作用地点：

Target.java

ElementType[]是一个数组，所以作用地点可以有多个

点进去看，有哪些地方可以用

@Documented – @Documented 所标注内容，可以出现在javadoc中。

@Inherited – @Inherited只能被用来标注“Annotation类型”，它所标注的Annotation具有继承性
